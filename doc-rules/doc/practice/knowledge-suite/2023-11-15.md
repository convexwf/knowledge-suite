# knowledge-suite Journal - 2023-11-15

## Saved tab: show collections as single entries, filter out collection clips

The Saved tab in the side panel previously showed ALL individual clips including those
saved via batch jobs. Collections now appear as single compact cards, and individual
clips that belong to any collection are filtered out.

Changes:
- `loadSavedClips()` now fetches both clips and collections in parallel.
- Builds a set of docIds/URLs that belong to any collection by fetching each
  collection's detail.
- Filters out clips that belong to collections → only standalone clips shown.
- New `renderSavedList(collections, standaloneClips)` shows collection cards
  (with icon, title, page count, created date) before standalone clips.
- Collection cards navigate to `items.html` on click.

```text
feat: show collections as single entries in saved tab

- loadSavedClips fetches clips + collections in parallel
- filter out clips that belong to any collection
- render collection cards before standalone clips
- collection card navigates to items page
```

## Fix: Saved tab collection cards UI + filtering

**Issue 2**: Collection cards in Saved tab now reuse the exact same `saved-item` / `saved-card-button` DOM structure as regular clip cards. Click navigates to the first sub-page (fetching collection detail), and a "First" button opens the first page directly.

**Issue 3**: Replaced the N+1 collection detail fetches with a single `GET /api/collections/used-doc-ids` call that returns all docIds referenced by `collection_items`. Filtering is now a single Set lookup. Added `getUsedCollectionDocIds()` in store and `usedCollectionDocIds()` in API client.

```text
fix: saved tab collection cards UI and filtering

- collection cards reuse saved-item/saved-card-button structure
- click opens first sub-page, First button opens first page
- add GET /api/collections/used-doc-ids endpoint
- replace N+1 filtering with single API call
```
