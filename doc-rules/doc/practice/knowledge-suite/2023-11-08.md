# knowledge-suite Journal - 2023-11-08

## Fix: collection navigation uses itemId to preserve reader context

When navigating prev/next in a collection, the navigation API only returned `docId`. The reader would load with `reader.html?docId=xxx`, losing the `itemId` context. This caused the collection nav buttons to work transiently but the page loaded without full item context.

Fix:

- Updated `getCollectionNavigation()` queries to LEFT JOIN `knowledge_items` on `active_doc_id`, returning both `doc_id` and `item_id`.
- Navigation response now includes `itemId` alongside `docId`.
- `navigateInCollection()` prefers `itemId` when available, falling back to `docId`.
- Now each navigation opens `reader.html?itemId=xxx`, preserving full reader context (reparse, annotations, collection display all work).
