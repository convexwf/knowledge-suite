# knowledge-suite Journal - 2023-10-20

## 16:48:09 fix: strip section anchors in preview and skip list-child deduplication

User reported two issues with document d47582aa clipped from clipper:

1. `<!-- section_id:s-de25b261b9 -->` HTML comments visible in preview panel
   but not in reader. Root cause: preview uses markdown-it with `html: false`,
   which escapes raw HTML instead of stripping it. Reader has its own
   line-by-line parser that correctly skips these comment lines (reader.ts:260).
   Fix: added `stripSectionAnchors()` in render.ts that removes lines matching
   `/^<!--\s*section_id:\S+\s*-->\n/gm` before passing markdown to markdown-it.

2. List items duplicated: appearing both in a list section and as standalone
   paragraph sections. Root cause: `extractSections()` uses `querySelectorAll`
   with deep descendant search across the whole DOM tree. When `<li>` contains
   child elements like `<p>` or `<blockquote>`, those descendant elements are
   also matched and processed as separate sections alongside their parent list.
   Fix: added `isInsideListItem()` check alongside the existing
   `isInsideMediaContainer()` check to skip any element whose ancestor is
   `<li>`.

Files changed:
- apps/knowledge-web-clipper/src/markdown-preview/render.ts: +stripSectionAnchors
- apps/knowledge-ingest-server/src/parser.ts: +isInsideListItem
