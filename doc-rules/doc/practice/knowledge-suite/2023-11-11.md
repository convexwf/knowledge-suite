# knowledge-suite Journal - 2023-11-11

## Fix: duplicate figure images when <img> is inside <figure>

When a page uses `<figure><img ...></figure>` (common pattern), the DOM-based parser extracted the same image twice:

1. `<figure>` element matched `SECTION_NODE_SELECTOR` → `figureSectionFromNode()` creates one figure section.
2. `<img>` inside the figure also matched the selector → `figureSectionFromNode()` creates a second figure section.

The existing `isInsideMediaContainer()` only checked for `<div>` parents, not `<figure>` or `<picture>`. Added `isInsideMediaTag()` that skips `<img>` elements whose direct parent is `<figure>` or `<picture>`.

This fixes all seven image pairs in the Xiaolinnote article (02_message_vs_shared_state, 03_shared_state_layers, 04_static_vs_dynamic_routing, etc.).
