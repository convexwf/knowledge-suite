# knowledge-suite Journal - 2023-11-04

## Fix: replace preview inline batch with modal overlay

User reported that replacing the preview area with the batch confirmation panel had no way to return to preview. Changed to a modal overlay approach:

- **Modal overlay**: Added `#batch-modal` with a semi-transparent overlay and bottom-sheet content panel. Batch content renders in the modal body, while the preview stays visible underneath.
- **Dismiss on overlay click**: Clicking outside the modal or the close button dismisses it.
- **Cancel running job**: Dismissing during an active batch job calls `cancelBatchJob` API (best-effort) and stops polling.
- **API client**: Added `cancelBatchJob()` method.
- **CSS**: Added `.batch-modal`, `.batch-modal-overlay`, `.batch-modal-content`, `.batch-modal-header`, `.batch-modal-body` styles as a bottom-sheet.
- **Removed**: `batchActive` flag, `hideAllViews()` function, all `renderInlineBatch*` functions replaced with `renderBatch*` modal equivalents.
